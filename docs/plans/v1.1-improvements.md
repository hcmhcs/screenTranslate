# ScreenTranslate v1.1 개선 기획서

> 작성일: 2026-03-02
> 수정일: 2026-03-02 (v1.0.1 핫픽스 완료, 코드 검증 반영)
> 기반: v1.0.1 (Phase 1~5 완료 + 핫픽스) 실사용 피드백

---

## 배경

v1.0 실사용 테스트에서 다음 개선 필요사항이 확인됨:
- 번역 결과를 다시 확인할 방법이 없음
- 언어팩 미설치 시 사용자가 상태를 알 수 없음
- 설정창/팝업 크기가 작아 내용이 잘림
- 원문과 번역문을 함께 비교할 수 없음
- 연속 번역 시 이전 요청이 에러로 표시됨

## 개선 항목

### 1. 번역 히스토리

**목적**: 이전 번역 결과를 다시 확인하고 활용할 수 있게 한다.

**요구사항**:
- 번역 완료/실패 기록을 시간순으로 저장
- 각 기록에 포함할 정보:
  - 타임스탬프
  - 원문 텍스트
  - 번역된 텍스트 (성공 시)
  - 실패 사유 (실패 시)
  - 소스 언어 → 타겟 언어
- 메뉴바에서 "히스토리" 항목으로 접근
- 히스토리 항목 클릭 시 번역문 복사
- 히스토리 전체 삭제 기능

**저장 방식**:
- v1.1: SwiftData 사용 (영구 저장, 최대 50건 표시)
- SwiftData를 사용하면 코드가 간단하고 향후 iCloud 동기화 가능

**데이터 모델** (SwiftData):
```swift
import SwiftData

@Model
final class TranslationRecord {
    var id: UUID                      // @Attribute(.unique) 불필요 - UUID는 이미 고유
    var timestamp: Date
    var sourceText: String
    var translatedText: String?       // nil이면 실패
    var failureReason: String?        // nil이면 성공
    var sourceLanguageCode: String    // "auto" 또는 언어 코드
    var targetLanguageCode: String

    init(
        sourceText: String,
        translatedText: String?,
        failureReason: String?,
        sourceLanguageCode: String,
        targetLanguageCode: String
    ) {
        self.id = UUID()
        self.timestamp = Date()
        self.sourceText = sourceText
        self.translatedText = translatedText
        self.failureReason = failureReason
        self.sourceLanguageCode = sourceLanguageCode
        self.targetLanguageCode = targetLanguageCode
    }
}
```

**⚠️ 아키텍처 주의사항**:
SwiftData `@Model`은 암시적으로 `@MainActor` 바인딩됩니다. 따라서 **Actor 기반 Manager는 사용하지 말고**, `@MainActor` 클래스를 사용해야 합니다:

```swift
// ❌ 잘못된 설계 (Actor + SwiftData = 안티패턴)
// actor TranslationHistoryManager { ... }

// ✅ 올바른 설계 — 프로젝트 전체가 @Observable 사용
@MainActor
@Observable
final class TranslationHistoryManager {
    static let shared = TranslationHistoryManager()
    private var modelContext: ModelContext?

    func configure(with context: ModelContext) {
        self.modelContext = context
    }

    func record(
        sourceText: String,
        translatedText: String?,
        failureReason: String?,
        sourceCode: String,
        targetCode: String
    ) throws {
        guard let context = modelContext else { return }
        let record = TranslationRecord(
            sourceText: sourceText,
            translatedText: translatedText,
            failureReason: failureReason,
            sourceLanguageCode: sourceCode,
            targetLanguageCode: targetCode
        )
        context.insert(record)
        try context.save()
    }

    func fetchRecent(limit: Int = 50) throws -> [TranslationRecord] {
        guard let context = modelContext else { return [] }
        var descriptor = FetchDescriptor<TranslationRecord>(  // ⚠️ var 필수 (fetchLimit 설정)
            sortBy: [SortDescriptor(\.timestamp, order: .reverse)]
        )
        descriptor.fetchLimit = limit
        return try context.fetch(descriptor)
    }

    func deleteAll() throws {
        guard let context = modelContext else { return }
        try context.delete(model: TranslationRecord.self)
        try context.save()
    }
}
```

**통합 지점** (기존 코드와의 연결):

⚠️ `record()`는 동기 메서드 (async 아님) — `await` 불필요.
⚠️ 실패 기록을 위해 `ocrResult`를 do 블록 밖으로 추출 필요.

```swift
// TranslationCoordinator.swift — startProcessing() 내부 구조 변경
func startProcessing(image: CGImage) {
    // ...
    currentTask = Task {
        // 실패 시에도 ocrResult에 접근하기 위해 do 블록 밖에 선언
        var ocrText: String?

        do {
            let ocrResult = try await ocrProvider.recognize(image: image)
            ocrText = ocrResult.text  // 실패 기록용 보관
            // ... (기존 로직)

            // :89 완료 시 히스토리 기록
            state = .completed(result)
            try? TranslationHistoryManager.shared.record(
                sourceText: ocrResult.text,
                translatedText: translated,
                failureReason: nil,
                sourceCode: effectiveSource?.minimalIdentifier ?? "auto",
                targetCode: targetLanguage.minimalIdentifier
            )
        } catch is CancellationError {
            state = .idle  // 기록하지 않음
        } catch OCRError.noTextFound {
            state = .failed("선택한 영역에서 텍스트를 찾을 수 없습니다.")
            // OCR 실패 — sourceText 없음, 기록 선택적
        } catch {
            state = .failed(error.localizedDescription)
            try? TranslationHistoryManager.shared.record(
                sourceText: ocrText ?? "(인식 실패)",
                translatedText: nil,
                failureReason: error.localizedDescription,
                sourceCode: sourceLanguage?.minimalIdentifier ?? "auto",
                targetCode: targetLanguage.minimalIdentifier
            )
        }
    }
}
```

### 2. 언어팩 상태 표시 및 다운로드

**목적**: 사용자가 언어팩 설치 상태를 한눈에 파악하고, 미설치 언어를 쉽게 다운로드할 수 있게 한다.

**요구사항**:
- 설정의 언어 피커에서 각 언어의 설치 상태 표시 (SF Symbols 사용)
  - `checkmark.circle.fill` (녹색): 다운로드됨 (즉시 사용 가능)
  - `arrow.down.circle` (주황색): 미다운로드 (다운로드 필요)
  - `ProgressView`: 다운로드 중
  - `xmark.circle` (회색): 지원하지 않음
- 미설치 언어를 선택하면 다운로드 확인 다이얼로그 표시
- 다운로드 진행률: 시스템 자체 UI 사용 (앱 커스텀 불가)

**Apple API**:

⚠️ `LanguageAvailability.status(from:to:)`는 **언어 쌍** 기반 API임.
개별 언어의 상태를 단독으로 확인할 수 없음.

**상태 확인 전략**: 현재 설정된 원문 언어를 기준으로 각 타겟 언어의 상태를 확인.
원문이 "자동 감지"인 경우 기준 언어를 `en`으로 대체하여 확인 (대부분의 쌍 커버).

```swift
import Translation

let availability = LanguageAvailability()

// 기준 언어: 원문 설정값, "auto"이면 "en" 대체
let baseLang = sourceLanguage ?? Locale.Language(identifier: "en")

// 각 타겟 언어별 상태 확인
for lang in AppSettings.supportedLanguages {
    let target = Locale.Language(identifier: lang.code)
    let status = await availability.status(from: baseLang, to: target)
    // status: .installed, .supported (다운로드 필요), .unsupported
}
```

### 3. 윈도우 크기 개선

**목적**: 설정창과 번역 팝업의 가독성을 높인다.

**현재 문제**:
- 설정창 (400×280): 항목이 많아지면 스크롤 발생
- 번역 팝업 (400×300 고정): 짧은 텍스트에는 과도하게 크고, 긴 텍스트에는 부족

**개선안**:

| 윈도우 | 현재 | 개선 |
|--------|------|------|
| 설정창 | 400×280 고정 | minWidth: 450, idealWidth: 500, maxWidth: 600<br>minHeight: 350, idealHeight: 400, maxHeight: 500 |
| 번역 팝업 | 400×300 고정 | 너비 320~480, 높이 콘텐츠 기반 (최소 120, 최대 400) |

**설정창 구현** (HIG 준수):

현재 설정 항목이 적으므로 (피커 4개 + 단축키 1개) 단일 Form 유지.
Phase 9 (언어팩 관리) 추가 후 항목이 많아지면 TabView 분리 검토.

```swift
// Settings scene 사용 (Cmd+, 자동 연결)
Settings {
    SettingsView()
        .frame(
            minWidth: 450, idealWidth: 500, maxWidth: 600,
            minHeight: 320, idealHeight: 380, maxHeight: 500
        )
}
```

**번역 팝업 크기 로직**:
```
- 로딩 상태 (인식 중/번역 중): 최소 크기 (320×120)
- 완료 상태: 텍스트 길이에 따라 유동적
  - 1~2줄: 320×160
  - 3~5줄: 400×250
  - 6줄 이상: 480×400 (스크롤)
- 에러 상태: 320×180
```

**팝업 크기 애니메이션** (HIG 준수):

⚠️ `withAnimation`은 SwiftUI 전용 — `NSWindow.setContentSize()`에는 적용 불가.
AppKit 네이티브 애니메이션인 `NSAnimationContext`를 사용해야 함.

```swift
func updateState(_ state: TranslationCoordinator.State, ...) {
    let newSize = calculateSize(for: state)

    // ✅ AppKit 네이티브 애니메이션
    NSAnimationContext.runAnimationGroup { context in
        context.duration = 0.2
        context.timingFunction = CAMediaTimingFunction(name: .easeInEaseOut)
        self.animator().setContentSize(newSize)
    }

    hostingView?.rootView = makePopupView(state: state)
}

// Phase 6: translatedText만 기반으로 크기 계산 (sourceText는 Phase 7에서 추가)
private func calculateSize(for state: TranslationCoordinator.State) -> NSSize {
    switch state {
    case .recognizing, .translating:
        return NSSize(width: 320, height: 120)
    case .completed(let result):
        let textHeight = estimateTextHeight(result.text)  // Phase 6: text만
        let height = min(max(textHeight + 80, 160), 400)
        let width = textHeight > 200 ? 480 : (textHeight > 100 ? 400 : 320)
        return NSSize(width: width, height: height)
    case .failed:
        return NSSize(width: 320, height: 180)
    case .idle:
        return .zero
    }
}
```

⚠️ `calculateOrigin()`도 동적 크기에 대응 필요:
현재 `popupHeight: 300`, `popupWidth: 400`이 하드코딩되어 있으므로
`calculateSize()`의 결과를 `calculateOrigin()`에 전달하도록 시그니처 변경 필요.

**HIG 준수 사항**:
- 크기 변경 시 `NSAnimationContext` 사용 (AppKit 네이티브)
- 텍스트 영역: I-beam 커서 (`NSCursor.iBeam`)
- `idealWidth`/`idealHeight` 파라미터 활용

**구현 위치**:
- `SettingsView` — frame 크기 확장
- `TranslationPopupWindow` — 동적 크기 계산 로직, 애니메이션 추가, `calculateOrigin` 시그니처 변경
- `TranslationPopupView` — idealWidth/maxHeight 조정

### 4. 원문 보기 기능

**목적**: 번역문과 원문을 나란히 비교할 수 있게 한다.

**요구사항**:
- 번역 완료 상태에서 "원문 보기" 토글 추가
- 토글 ON: 원문 (위) + 구분선 + 번역문 (아래)
- 토글 OFF: 번역문만 표시 (기본값)
- 원문도 텍스트 선택 + 복사 가능
- 키보드 단축키: `Cmd+O`

**데이터 변경**:

```swift
// 기존
struct TranslationResult: Sendable {
```

**UI 구현** (HIG 준수):

```swift
struct TranslationPopupView: View {
    @State private var showingOriginal = false

    var body: some View {
        VStack(alignment: .leading, spacing: 12) {
            // 번역문
            ScrollView {
                Text(result.translatedText)
                    .font(.body)
                    .foregroundStyle(.primary)
                    .textSelection(.enabled)
            }

            // 원문 (토글 시)
            if showingOriginal {
                Divider()

                VStack(alignment: .leading, spacing: 4) {
                    HStack {
                        Text("원문")
                            .font(.caption)
                            .foregroundStyle(.secondary)
                        Spacer()
                        // ⚠️ Locale.Language에 localizedName 프로퍼티 없음
                        if let lang = result.sourceLanguage {
                            Text(Locale.current.localizedString(
                                forIdentifier: lang.minimalIdentifier) ?? "")
                                .font(.caption)
                                .foregroundStyle(.secondary)
                        }
                    }
                    Text(result.sourceText)
                        .font(.body)
                        .foregroundStyle(.secondary)
                        .textSelection(.enabled)
                }
                .transition(.opacity.combined(with: .move(edge: .top)))
            }

            // 버튼
            HStack {
                Toggle("원문 보기", isOn: $showingOriginal)
                    .toggleStyle(.checkbox)
                    .keyboardShortcut("o", modifiers: .command)

                Spacer()

                Button("복사") { onCopy(result.translatedText) }  // 항상 번역문만
                    .keyboardShortcut("c", modifiers: .command)

                Button("닫기") { onClose() }
                    .keyboardShortcut(.cancelAction)
            }
        }
        .animation(.easeInOut(duration: 0.2), value: showingOriginal)
    }
}
```
